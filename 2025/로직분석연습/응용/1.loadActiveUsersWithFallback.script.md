loadActiveUsersWithFallback 함수 
1. API 를 호출하는 함수
2. defaultUsers 변수는 디폴트 즉 가져온 API 값이 없을때 기본적으로 노출될 수 있는 데이터로 보여짐

3. 매개변수로 받아온 apiClient 를 통해서 api/users 를 호출 
 - 성공 시 res 값을 가지고 조건문 수행 
 - 만약에 res 값이 없거나, res.status의 값이 200이 아니면 미리 정의한 defaultUsers 반환
 - 만약에 res.data 값이 배열이 아니면 defaultUsers 반환
 - actives 빈배열 선언
 - for...of 반복문 실행 (개체는 아님) res.data 의 값을 가지고 나와서 u에 담아주면서 반복
    - 만약에 u 가 없거나, u.active의 값이 없으면 (둘다 없어야됨) 다음 반복문으로 현재 반복문은 넘어감
    - (둘 중 하나의 값이 있으면 다음 로직 이어서 진행)
    - level 변수에 u.grade 가 VIP와 동일하면 VIP 아니면 NORMAL 을 담는다.
    - 그리고 미리 선언한 빈 배열을 갖고 있는 actives에 객체를 push 해준다. 
    [{id: string(u.id), name: u.name || '이름없음', level: level}, ... ]
    name: u.name || '이름없음' 이거는 u.name 이 값이 없으면 '이름없음'이 대신 나올 수 있게 처리되어 있음
    - 다음 조건문에서 actives.length 길이가 0 이면 defaultUsers 값 반환 
    - 이상 없으면 반복문을 통해 데이터들을 actives에 담아 반환 한다.
    
 - for...of 반복문 중에 error 가 발생하게 되면 catch 문을 통해서 error 로그를     console.log를 통해 err, message 를 보여주고 defaultUsers 값으로 반환 시켜 준다.
 -> 이렇게 하는 이유는 일단 화면을 보여주는게 중요하기 때문임 아무것도 나오지 않으면 고객 입장에서 더한 불만이 나오기 때문 일단 화면을 보여주고 이후 문제를 수정해 나가는게 사용자 관점에서 유용함

 -> 수저: then 체인 전체에서 에러가 난다면 마지막 .catch에서 로그를 남기고 defaultUsers를 반환한다.



/**
 * loadActiveUsersWithFallback 함수
 *
 * 1. 하는 일
 *    - /api/users 를 호출해서 "활성화된(active) 유저 목록"을 가져오되,
 *      응답이 이상하거나 에러가 나면 defaultUsers(게스트 정보)를 대신 반환한다.
 *
 * 2. 매개변수
 *    - apiClient: URL을 받아서 { status, data } 형태의 응답을 Promise로 돌려주는 함수.
 *
 * 3. 기본값
 *    - defaultUsers: API가 실패했을 때 화면에 최소한으로 보여줄 수 있는 기본 유저 배열.
 *
 * 4. 정상 흐름
 *    - apiClient('/api/users') 호출
 *    - res가 없거나, res.status !== 200 이면 defaultUsers 반환
 *    - res.data가 배열이 아니면 defaultUsers 반환
 *    - res.data를 for...of로 순회하면서:
 *        - u가 없거나 u.active가 false면 continue (건너뜀)
 *        - u.grade가 'VIP'면 level = 'VIP', 아니면 'NORMAL'
 *        - actives 배열에 { id: String(u.id), name: u.name || '이름없음', level } 형태로 푸시
 *    - actives.length가 0이면 defaultUsers 반환
 *    - 그렇지 않으면 actives 반환
 *
 * 5. 에러 흐름
 *    - 위 과정 중 어느 지점에서든 에러가 나면 .catch로 들어옴
 *    - 에러 메시지를 로그로 찍고, defaultUsers를 반환
 *
 * 6. 의도
 *    - API가 망가져도 "아무것도 안 보이게" 만들지 않고,
 *      최소한의 기본 유저 정보라도 보여주기 위한 Fallback 처리.
 */
